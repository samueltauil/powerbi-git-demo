# Contributing to powerbi-git-demo

Thank you for your interest in contributing! This is a public educational repository that demonstrates Power BI and Microsoft Fabric Git integration. Contributions are welcome, particularly improvements to documentation, examples, and error fixes.

## How to Report Issues

If you find something incorrect, unclear, or out of date:

1. Open a [GitHub Issue](https://github.com/samueltauil/powerbi-git-demo/issues) with a clear title and description
2. Include the section or file that needs attention
3. If applicable, include the correct information or a suggested fix

## How to Suggest Improvements

If you have an idea for a new section, a clearer explanation, or an additional example:

1. Open an issue first to discuss the idea before starting work
2. Fork the repository and create a feature branch
3. Make your changes and open a Pull Request with a clear description of what was changed and why

## What to Contribute

Because this is primarily a **documentation and demo** repository, contributions should focus on:

- **Documentation clarity** — improving explanations, fixing grammar or typos, adding helpful context
- **Additional examples** — new diagrams, screenshots, or walkthroughs that illustrate concepts
- **Fixing errors** — correcting broken links, outdated information, or inaccurate instructions

> **Please do not modify** `.tmdl`, `.pbir`, `.pbip`, `.pbit`, `report.json`, `.platform`, or `.gitignore` files unless there is a specific documented reason to do so. Changes to these files affect the Power BI project structure and are out of scope for most contributions.

## Code of Conduct

Be respectful and constructive in all interactions. This is a learning resource — questions and beginner contributions are welcome.
